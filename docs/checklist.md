# 模块文档检查清单

基于 `modules/ctle.md` 标准结构。每个模块文档必须通过以下所有检查项。

---

## 文档头部

- [ ] 包含模块级别（AMS/DE子模块）
- [ ] 包含类名
- [ ] 包含当前版本号和日期
- [ ] 包含状态（开发中/生产就绪）

---

## 1. 概述

- [ ] **1.1 设计原理**：说明模块的核心设计思想和工作原理
- [ ] **1.2 核心特性**：列出模块的主要功能特性（bullet list）
- [ ] **1.3 版本历史**：版本号、日期、主要变更（表格形式）

---

## 2. 模块接口

- [ ] **2.1 端口定义**：端口名、方向、类型、说明（表格形式）
- [ ] **2.2 参数配置**：
  - [ ] 基本参数表格（参数名、类型、默认值、说明）
  - [ ] 子结构参数说明（如有）
  - [ ] 每个子结构的工作原理简述

---

## 3. 核心实现机制

- [ ] **3.1 信号处理流程**：处理步骤的流程图或文字描述
- [ ] **3.2 关键算法/机制**：传递函数构建、滤波器设计等
- [ ] **3.3 设计决策说明**：为什么这样设计（如软饱和选择tanh的原因）

---

## 4. 测试平台架构

- [ ] **4.1 测试平台设计思想**：场景驱动、组件复用等设计理念
- [ ] **4.2 测试场景定义**：场景名、命令行参数、测试目标、输出文件（表格）
- [ ] **4.3 场景配置详解**：每个场景的详细配置和验证点
- [ ] **4.4 信号连接拓扑**：ASCII图或文字描述模块连接关系
- [ ] **4.5 辅助模块说明**：信号源、监控器等辅助模块的功能说明

---

## 5. 仿真结果分析

- [ ] **5.1 统计指标说明**：均值、RMS、峰峰值等指标定义
- [ ] **5.2 典型测试结果解读**：各场景的期望结果和分析方法
- [ ] **5.3 波形数据文件格式**：输出文件格式说明

---

## 6. 运行指南

- [ ] **6.1 环境配置**：环境变量设置
- [ ] **6.2 构建与运行**：cmake/make命令、场景参数
- [ ] **6.3 结果查看**：Python脚本或其他可视化方法

---

## 7. 技术要点

- [ ] 列出关键技术决策和注意事项
- [ ] 说明已知限制或特殊要求
- [ ] 与相关模块的对比（如适用）

---

## 8. 参考信息

- [ ] **8.1 相关文件**：源码、测试、脚本路径（表格）
- [ ] **8.2 依赖项**：SystemC版本、其他库
- [ ] **8.3 配置示例**：JSON配置代码块

---

## 文档尾部

- [ ] 文档版本号
- [ ] 最后更新日期
- [ ] 作者
